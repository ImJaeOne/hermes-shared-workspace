# 영업 파이프라인 플레이북

> 템플릿: `sales_pipeline_v1`
> 에이전트: Sales Agent

## 파이프라인 개요

```
Lead In → Qualification → Proposal(승인) → Negotiation → Close(승인)
```

## 단계별 가이드

### 1. Lead In (리드 유입)

**목표**: 잠재 고객 정보를 정리하고 초기 접촉 여부를 판단한다.

**체크리스트**:
- [ ] 고객 기본 정보 수집 (이름, 회사, 직급, 연락처)
- [ ] 유입 경로 확인 (컨퍼런스, 웹사이트, 소개 등)
- [ ] 고객사 규모 및 산업군 파악
- [ ] 초기 니즈 메모

**산출물**: `contact_info` (JSON)

**전환 조건**: 연락처 정보 확보 완료

---

### 2. Qualification (검증)

**목표**: 리드의 적합도를 평가하고, 미팅을 통해 구체적 니즈를 파악한다.

**체크리스트**:
- [ ] 초기 연락 이메일 발송
- [ ] 고객 응답 확인
- [ ] 미팅 일정 조율 및 진행
- [ ] BANT 검증 (Budget, Authority, Need, Timeline)
- [ ] 미팅 노트 정리 및 공유

**산출물**: `email`, `meeting_notes`

**전환 조건**: 미팅 완료 + 고객 니즈 구체화

---

### 3. Proposal (제안) — 승인 필요

**목표**: 고객 맞춤 솔루션 제안서를 작성하여 가치를 전달한다.

**체크리스트**:
- [ ] 고객 요구사항 정리
- [ ] 솔루션 매핑 (제품/서비스 → 고객 과제)
- [ ] 가격 구성 확정
- [ ] 제안서 초안 작성
- [ ] 내부 리뷰 (팀장 검토)
- [ ] 제안서 발송 이메일 작성

**산출물**: `proposal`, `email`

**승인**: 매니저 승인 필요 → 제안서 품질 및 가격 정책 검증

---

### 4. Negotiation (협상)

**목표**: 계약 조건을 협의하고 합의점을 도출한다.

**체크리스트**:
- [ ] 고객 피드백 수집 (가격, 조건, 범위)
- [ ] 협상 포인트 정리
- [ ] 계약서 초안 작성
- [ ] 법무 검토 요청 (필요 시)
- [ ] 수정안 반영

**산출물**: `contract`, `email`

**전환 조건**: 양측 합의 완료

---

### 5. Close (계약) — 승인 필요

**목표**: 최종 계약을 체결하고 온보딩을 준비한다.

**체크리스트**:
- [ ] 최종 계약서 확정
- [ ] 서명 절차 진행
- [ ] 계약 완료 보고서 작성
- [ ] 온보딩 팀 인계

**산출물**: `contract`, `report`

**승인**: 매니저 + 디렉터 승인 필요

## 핵심 원칙
- 모든 커뮤니케이션은 산출물로 기록한다
- 각 단계의 산출물이 완료되어야 다음 단계로 전환한다
- 승인 단계에서 거절되면 피드백을 반영하여 산출물을 보완한다
- 고객 응대 이메일은 24시간 내 발송한다
